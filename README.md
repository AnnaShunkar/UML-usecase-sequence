# UML-usecase-sequence
# Пояснення послідовності дій
- Користувач через UI обирає фільм, сеанс і місце.
- UI запитує дані про сеанс у ShowService, який звертається до бази даних.
- Потім перевіряється доступність місця через SeatService.
- Якщо місце доступне, BookingService створює бронювання, записуючи його в базу.
- Користувач отримує підтвердження та квиток.
